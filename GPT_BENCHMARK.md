# GPT Benchmark: AI Assistant 산출물 비교

세 AI 어시스턴트(Claude, Codex/GPT, Gemini)가 작성한 문서를 세트별로 비교 정리했다. 각 문서의 강점/약점/사용처를 한글로 상세히 기술한다.

## 요약 스냅샷
- Claude: 방대한 맥락·리스크 중심, 아키텍처 설명이 깊음.
- Codex/GPT: 간결·실무 지향, 빠른 온보딩에 적합.
- Gemini: 5분 요약형 온보딩, 일부 내용 구버전 혼재.

## 세트별 상세 비교
### Claude 세트 (CLAUDE.md, RISK_ANALYSIS.md, QUALITY_REPORT.md)
- 장점
  - 아키텍처/도메인/흐름을 매우 깊이 서술. 수집기 분리, 상태 전이, 보안·동시성 맥락을 모두 포함.
  - RISK_ANALYSIS.md가 우선순위·ETA·해결 효과를 명시한 백로그 형태라, 하드닝 계획 수립에 유용.
  - collectionRequestedAt 추가, @Lock 위치 수정 등 최신 변경사항을 근거와 함께 기록.
  - 코드 블록과 경로, 구체적 쿼리/락 전략까지 적시하여 추적성 높음.
- 단점
  - 분량이 과다하고 반복 설명이 많아 빠른 검색성이 떨어짐.
  - 프로토타입 범위를 넘어 JWT/MQ/관측성 등 프로덕션 권고가 섞여 스코프 혼동 가능.
  - “코드 스멜 개수” 같은 메트릭은 지속 관리 없으면 바로 최신성이 떨어짐.
- 활용처
  - 전사적 감사/리스크 리뷰, 하드닝 로드맵 설계 시 참고용 레퍼런스.
  - 신규 기여자보다는 테크리드/아키텍트용 딥다이브 자료.

### Codex/GPT 세트 (AGENTS.md, GPT_RISK_ANALYSIS.md, GPT_QUALITY_REPORT.md)
- 장점
  - AGENTS.md가 리포 규칙·빌드/테스트/런 명령·모듈 책임을 짧게 정리해 온보딩 속도가 빠름.
  - GPT_RISK_ANALYSIS.md는 핵심 위험에 집중(헤더 인증 스푸핑, 5분 블로킹, 권한 검증 누락, 폴링 부하, Excel 실패 처리, 메모리 페이징 등)하여 실무 액션 아이템으로 바로 사용 가능.
  - GPT_QUALITY_REPORT.md는 강점/갭을 균형 있게 요약하며 최근 테스트 상태 주석 포함.
  - 전체적으로 간결해 업데이트 부담이 낮음.
- 단점
  - 각 위험에 대한 ETA/난이도/우선순위가 구체적이지 않음.
  - 일부 진척 상황(예: collector 리팩터 후 상태)이 암묵적으로 전제되어 초심자에게는 부족할 수 있음.
  - 해결 완료 항목의 로그/히스토리 정리가 얕음.
- 활용처
  - 일상 개발 레퍼런스, 빠른 체크리스트. PR 준비나 단기 스프린트 계획 시 “해야 할 것”을 바로 뽑을 때 적합.

### Gemini 세트 (GEMINI.md, GEMINI_RISK_ANALYSIS.md, GEMINI_QUALITY_REPORT.md)
- 장점
  - 5분 온보딩 가이드 형식으로 모듈, 주요 엔드포인트, 실행/테스트 명령을 빠르게 파악 가능.
  - 모듈 책임, 아키텍처, VAT 계산 규칙, H2 접속 정보 등을 짧게 정리.
  - 리스크/퀄리티 리포트가 가독성 높은 내러티브로 작성됨.
- 단점
  - 일부 내용이 최신 코드와 불일치 (@Transactional + 5분 sleep 유지 등 과거 상태 언급).
  - 우선순위/ETA 부재, 구체적 조치보다는 개괄적 권고가 많음.
  - “이미 수정” 언급이 실제 코드 상태와 다를 수 있어 교차 확인 필요.
- 활용처
  - 첫 읽기용 프라이머. 최신 상태 확인을 전제로 참고해야 함.

## 교차 권고
1) 기준 문서 정리: AGENTS.md를 공식 온보딩·규칙서로 유지하고, 위험 목록은 Claude/GPT 내용을 취사선택해 단일 우선순위 표로 관리.
2) 최신화: Gemini 세트는 collector 리팩터(비동기 분리, @Lock 리포지토리 적용, `collectionRequestedAt` 추가 등) 반영해 갱신.
3) 사용 가이드: 단기 액션은 GPT_RISK_ANALYSIS.md 기반, 장기·심층 검토는 RISK_ANALYSIS.md 참고, 신규 인원은 GEMINI.md로 개략 파악 후 AGENTS.md와 최신 리스크 표를 확인.
